# Training Runs

Per-checkpoint data configuration. Structure:

    runs/<model>/<run_name>/
        build.sh        — script to reproduce the training data
        train.jsonl      — training data (symlink or generated by build.sh)
        val.jsonl        — validation data (symlink or generated by build.sh)

Each build.sh documents exactly how the training data was produced:
- Which shared datasets were used (references, symlinks, or concatenation)
- Any run-specific data generation (custom sampling, filtering, etc.)
- Ratios and mixing strategy
